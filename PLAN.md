# 소방관 전설이 되다 — 설계 플랜

> 이 문서는 Claude와의 개발 세션에서 정리된 설계 플랜이다. 기기가 바뀌어도(PC ↔ 태블릿 등)
> 이 문서를 기준으로 이어서 작업한다. 리포에 있는 `DESIGN.md`는 별도 세션에서 먼저 작성됐던
> 초기 아이디어 문서로, 현재는 **이 PLAN.md 쪽 설계를 기준으로 진행 중**이다(참고용 역사 기록으로만 남김).

## 현재 상태 (진행 상황)

- **1단계(세로 슬라이스) 완료**: 마을/소방서(HQ)/"우리 집" 던전 씬, 타일 이동, 불씨요정 대화,
  화재이벤트, 몬스터 전투(화재등급 상성 포함), 퀘스트 체인 6단계, 장비상점, HQ 티어업,
  세이브/로드까지 브라우저에서 엔드투엔드로 동작 확인 완료.
- **학습 시스템 개선 완료**: 레슨을 던전 진입 시 한꺼번에 주지 않고 인카운터마다 조금씩 나눠
  전달(자동 균등 배분), 챕터(던전) 클리어 시 "🎖️ 진급시험"(문제 8개 몰아서 풀기 + 등급/보상)
  자동 트리거하는 구조로 완성.
- **다음: 2단계(콘텐츠 확장)** — 아래 "단계별 진행" 참고.

## Context

플레이타임 약 5시간을 목표로 하는 소방교육 퀴즈 기반 RPG를 신규로 제작한다. 참고용으로 공유받은
"모여봐요 과학의숲"(사용자의 기존 프로젝트, 5422줄 단일 HTML)은 정적 탭 전환형 농장 게임이었으나,
이번 프로젝트는 완전히 다른 장르다: **캐릭터를 타일 단위로 직접 조작**하고, 씬(마을/소방서/화재현장)을
넘나들며, 포켓몬식 턴제 전투에 퀴즈를 직접 결합한 형태다. 공용 자산으로 쓸 실제 커리큘럼 자료
(`한국화재보험협회 2026불조심길라잡이.pdf`, 27판)를 확보해 텍스트를 전량 추출했다
(`content/curriculum_full_text.txt`, 저작권 있는 원본 발췌라 **`.gitignore`로 리포에서 제외** —
로컬 개발 참고용으로만 보관, 게임 자체는 이를 참고해 새로 쓴 `js/data-quiz.js` 콘텐츠만 사용).

사용자와 확정한 방향:
- 이동: **타일(칸) 단위**, 화살표 버튼(모바일) + 방향키(데스크톱)
- 맵: **씬 단위 분리** (마을/소방서/화재현장 던전을 별도 맵으로, 문·입구로 전환)
- 렌더링: **Canvas 기반**
- 아트: **이모지+CSS 우선**, 나중에 이미지 에셋 교체 가능한 구조로
- 퀴즈 문제 은행: **커리큘럼 원문 기반으로 직접 제작**
- 전투: **퀴즈가 전투에 직접 개입** (정답=강공격/치명타, 오답=몬스터 반격)
- Part 14 대형 화재사례(씨랜드/대구지하철 등 실제 참사, 사망자 다수): **추모관 콘텐츠로만 사용**,
  퀴즈·던전 소재로 게임화하지 않음

**중요한 전제**: 과학의숲은 학교에서 이미 배운 내용을 복습하는 게임이었다. 이 게임은 반대로,
아이들이 이 게임을 통해 **처음** 소방안전 지식을 습득하는 것을 목표로 한다. 따라서 "느닷없이
퀴즈부터 나오고 틀리면 맞는다" 구조는 부적합하며, 아래 "학습 시스템" 섹션이 퀴즈/전투 시스템보다
선행하는 핵심 설계 요소다.

## 게임 개요

- 제목: 소방관 전설이 되다
- 주인공: 별볼일 없는 신입, 무너져가는 창고(개인 소방서)를 함께 성장시킴
- 핵심 루프: 마을에서 퀘스트 수령 → 화재현장 던전 이동(타일 이동) → 화재이벤트 해결(퀴즈 1회성
  판단) 또는 불꽃 몬스터 전투(퀴즈 연동 턴제) → 자원/골드 획득 → 장비 강화 + 소방서(HQ) 확장 →
  새 지역·NPC·퀘스트 해금

### 캐릭터 훅 — 불씨요정 (마스코트/학습 가이드)
- 오프닝: 주인공 눈에만 보이는 작은 불씨요정(가칭)이 나타난다. 요정 자체가 위험을 경고하는 게
  아니라, 그냥 존재할 뿐인데 생김새가 영락없는 불꽃이라 주인공이 요정을 볼 때마다 소스라치게
  놀라 "불이야!!"하고 소리친다. 하지만 남들 눈엔 요정도, 불도 전혀 보이지 않으니 마을 사람들은
  주인공을 "허구한 날 헛것 보고 불났다 소리치는 이상한 애"로 취급한다 — 이것이 주인공이
  "별볼일 없는" 존재로 여겨지는 이유가 된다. (이후 요정이 실은 진짜 위험도 감지할 수 있다는
  설정으로 이어지며, 주인공의 "호들갑"이 사실은 진짜였다는 게 하나둘 밝혀지는 것이 초반 서사의
  재미 포인트)
- 요정은 게임 전체에서 **학습 시스템의 화자**를 맡는다: 레슨 대사, 퀴즈 정답/오답 해설, 안전 수첩의
  내용은 모두 요정이 알려주고 정리해주는 형태로 일관되게 전달된다 (딱딱한 NPC 설명 대신 친근한
  동반자 캐릭터가 가르쳐주는 톤 유지)
- 스토리 아크: 초반엔 요정 때문에 놀림받던 주인공이, 퀘스트를 해결하고 실제로 위험을 막아내는 걸
  마을 사람들이 하나둘 목격하면서 신뢰를 얻고, 결국 "전설이 된 소방관"으로 인정받는 성장 서사로
  이어진다 (제목과 직결되는 메인 퀘스트 체인의 감정적 축)

## 학습 시스템 (처음 배우는 콘텐츠 — 퀴즈보다 선행)

퀴즈/전투 시스템 설계의 전제가 되는 핵심 레이어. 모든 카테고리는 "배우기 → 적용하기 → 시험 →
복습 가능" 4단계로 구성한다.

1. **레슨(선행 학습, 조금씩 나눠서)**: 불씨요정과의 대화로 미니 강의를 전달하되, **던전에 처음
   들어갈 때 한꺼번에 다 주지 않는다.** 그 던전의 "전달 계기"(입장 1회 + 몬스터 처치·화재이벤트
   해결 등 인카운터마다 1회)에 맞춰 레슨을 균등 배분한다 — 예를 들어 레슨 7개·인카운터 4개(=계기
   5개)면 2-2-1-1-1개씩 나눠, 던전 입장 시 첫 토픽만 주고 몬스터를 한 마리 잡을 때마다 다음 토픽이
   이어지는 식이다("퀴즈 2~3문제 풀 분량씩 → 인카운터 한 건 → 다음 이야기"). 마지막 인카운터가
   끝나는 시점에 정확히 마지막 레슨까지 소진되도록 계기 수에 맞춰 배분량을 계산한다.
2. **적용(퀴즈/화재이벤트/전투)**: 레슨에서 다룬 내용을 그대로 퀴즈로 낸다 — 즉 문제 출제 범위는
   "방금 배운 것"으로 한정되어, 아이가 배운 걸 바로 써먹는 구조가 된다. 정답/오답 관계없이
   **문제를 푼 직후 반드시 해설 1~2줄을 보여준다**(정답이면 "맞아요! ~하기 때문이에요", 오답이면
   "아쉬워요, 정답은 ○○예요. 왜냐하면 ~"). 전투 중에도 이 해설 창은 데미지 연출 전에 짧게 노출한다.
   오답이 벌칙(몬스터 반격/이벤트 실패)으로 이어지더라도, 그 벌칙 자체가 "왜 틀렸는지" 알게 되는
   기회가 되도록 해설을 생략하지 않는다.
3. **시험(챕터 진급시험)**: 던전의 모든 몬스터/화재이벤트를 처리해 "챕터 클리어" 상태가 되면,
   그 챕터 문제 은행에서 8문제를 뽑아 "🎖️ 소방관 진급시험" 형태로 몰아서 출제한다(문제마다 해설은
   그대로 유지). 다 풀면 정답 수에 따라 등급(특급합격/합격/턱걸이/재도전 권장)과 보상(골드·자재
   보너스)을 주는 결과 화면을 보여준다. 실패해도 챕터 진행을 막지는 않고 복습을 유도하는 톤으로만
   안내한다.
4. **복습(안전 수첩)**: 지금까지 배운 레슨/퀴즈 해설이 카테고리별로 자동 축적되는 "안전 수첩" UI를
   상시 열람 가능하게 제공한다(도감 형태, "불씨요정이 정리해준 수첩"이라는 설정). 재도전하거나
   다른 던전에서 헷갈릴 때 언제든 되돌아볼 수 있어, "학습 → 게임 진행 중 반복 노출 → 진급시험 →
   자연스러운 암기"로 이어지게 한다.

레슨(토픽 단위)은 처음 전달될 때 스킵 불가이며, 던전 재방문 시 이미 다 받은 레슨은 다시 나오지
않는다(수첩에서 언제든 재열람 가능하므로). 이 구조 덕분에 퀴즈 문제 은행을 작성할 때도 "문제 카드"뿐
아니라 그 앞에 붙는 "레슨 카드"를 세트로 함께 작성해야 한다 (`js/data-quiz.js`의 각 카테고리는
`lessons: [...]`와 `questions: [...]` 두 배열로 구성).

## 전체 아키텍처

### 파일 구성
과학의숲처럼 완전 단일 파일로 가면 예상 분량(엔진+퀴즈뱅크 200문항+씬 데이터)이
8,000~12,000줄을 넘어 유지보수가 어려워진다. 대신 **일반 `<script src>` 방식**(ES 모듈 아님 →
file:// 더블클릭으로 그대로 실행 가능, CORS 문제 없음)으로 분리:

```
index.html          — 골격 HTML, CSS, 캔버스, UI 오버레이(퀴즈모달/전투UI/상점/퀘스트보드)
js/data-quiz.js      — QUIZ_BANK (카테고리별 레슨+문제 배열)
js/data-content.js   — ARMOR/TOOLS/ITEMS, MONSTERS, QUESTS, SCENES(마을/HQ/던전), HQ_TIERS 등 정적 데이터
js/engine-scene.js   — 타일맵 렌더링, 카메라, 충돌, 캐릭터 이동/애니메이션 (Canvas)
js/engine-battle.js  — 턴제 전투 루프 + 퀴즈 연동 + 화재등급 상성
js/game.js           — GameState, 학습 시스템, 퀴즈 UI, 상점/HQ/퀘스트, 세이브/로드, 부트스트랩
assets/              — (2단계) 이미지 에셋 자리, 현재는 비어있고 이모지로 대체
```

### 이동 & 씬 시스템
- 타일 그리드(40px/타일) 위에서 4방향 이동, 이동 애니메이션은 타일 간 보간(150ms)
- 씬 정의: `{ id, grid(문자열 배열: '#'=벽/'.'=바닥), doors, npcs, monsters, fireEvents }`
- 문/입구 타일에 닿으면 다음 씬의 지정 스폰 좌표로 이동
- 카메라: 캔버스 중앙에 플레이어 고정, 맵 가장자리에서 클램프
- 입력: 키보드 방향키 + 상시 노출 On-screen D-pad(모바일/터치), 타일 이동 중에는 입력 큐잉

### 전투 시스템 (포켓몬식 + 퀴즈 개입)
- 던전 내 몬스터 타일에 접촉 → 전투 화면 전환(별도 오버레이, 씬 일시정지)
- 매 턴 선택지: **[퀴즈로 공격] [아이템 사용] [도망가기]**
  - 퀴즈로 공격: 장착한 "전투 세트" 도구 중 하나 선택 → 해당 던전 카테고리(이미 레슨을 마친 범위)
    에서 문제 출제 → 정답 시 도구-몬스터 화재등급 상성(아래 장비 성장 섹션 참조)을 적용한 데미지
    (상성 적중=치명타급, 상성 어긋남=약화/역효과), 오답 시 즉시 몬스터 턴(플레이어 피격, DEF로
    경감). **정답/오답 판정 직후 데미지 연출 전에 1~2줄 해설을 항상 노출**(학습 시스템 참조,
    상성이 어긋났다면 "왜 이 도구가 이 불에는 안 맞는지"도 함께 설명) — 전투도 학습 기회여야
    하므로 해설 없이 넘어가지 않는다
  - 아이템 사용: 소비 아이템(응급키트=회복 등) 사용 후 몬스터 턴 진행
  - 도망가기: 확률 성공 시 던전 맵으로 복귀(보스전 제외)
- 몬스터 HP 0 → 승리 연출 + 자원/골드/경험치 지급, 플레이어 HP 0 → HQ로 후송(자원 소량 손실, 죽음 없음)
- 몬스터 데이터: 던전별 테마 몬스터 3~5종 (스파크도깨비/유류슬라임/가스망령/산불정령 등), 각 몬스터는
  화재등급 속성(A/B/C/K/연기)을 가지며 스탯은 던전 난이도에 따라 스케일링

### 화재이벤트 (전투와 별개, 1회성 판단형)
- 던전 내 특정 타일(예: 불타는 가전제품, 기름통, 가스레인지)에 접촉 시 즉시 퀴즈 1문항 제시
  (예: "유류화재에는 무엇을 쓰면 안 될까?" → 정답 시 이벤트 해결 + 보상, 오답 시 소량 피해 후
  이벤트 유지, 재시도 가능). 여기서도 판정 직후 해설이 항상 뜬다
- 커리큘럼의 "행동요령형" 지식(3대 소화법, 옷에 불 붙었을 때 STOP-DROP-ROLL 등)을 이 포맷으로 소화

### 캐릭터 & 장비 성장
- 스탯: HP/ATK/DEF/치명타%, 레벨업은 전투/퀘스트 경험치로 (HUD에 HP바 + EXP바 함께 표시)
- 방어구 슬롯 3종(헬멧/방화복/장갑): 단순 5티어 스탯 스케일링
- **특수장비(공격 도구)는 티어보다 "종류"가 핵심**: 분말(ABC)소화기, K급 소화기, 물호스(옥내소화전),
  담요/모래, 도끼, 물에 적신 수건 등 다양한 도구를 보유
- **화재등급 상성표**: 몬스터/화재이벤트마다 A급(종이·나무·천)/B급(유류)/C급(전기)/K급(식용유)/
  연기·질식형 중 하나의 속성을 가지며, 도구별 상성이 존재
  - 분말(ABC)소화기: A/B/C 전반에 무난 (초반 기본 지급 장비)
  - 물호스: A급에 특효, **C급(전기)에는 역효과**(감전 데미지를 오히려 자신이 받음),
    B급(유류)엔 비효율(데미지 대폭 감소)
  - K급 소화기: K급(주방·식용유)에 특효, 그 외엔 평범
  - 담요/모래(질식소화): A/B급에 유효(산소 차단), C급엔 비유효
  - 도끼: 전투 데미지는 고정(상성 없음) 대신 던전 기믹(문 부수기·장애물 제거·구조)의 필수 유틸리티
  - 젖은 수건: 전투용이 아니라 연기·질식형 화재이벤트(대피 판정)의 필수 아이템
  - 상성이 맞으면 큰 데미지, 안 맞으면 축소/역효과 — "왜 이 도구를 써야 하는지"를 퀴즈 해설과
    함께 체득하게 하는 장치
- 장비상점(마을): 골드+자재로 새 도구/방어구 구매, 상위 티어는 HQ 티어 해금에 연동

### 소방서(HQ) 성장 — 창고 → 초호화 소방본부
- 5티어: 폐창고 → 가건물 → 기초소방서 → 현대식소방서 → 초호화소방본부
- 티어업 조건: 누적 자재+골드 임계값 (메인 퀘스트 진행과 연동)
- 티어별로: 배경/외관 CSS 변경, 새 NPC 입주, 새 훈련시설(패시브 보너스) 슬롯 해금,
  장비상점 상위 티어 언락, **추모관**(Part 14) 콘텐츠는 3티어 이후 해금

### 퀘스트 시스템
- 마을 NPC 퀘스트보드 UI(목표/보상/납품 구조)
- 메인 퀘스트 체인이 던전 해금 순서를 결정 → "여러 단계를 거치는" 진행 구조
- 사이드 퀘스트는 특별편 1~4 시나리오 + 마을 주민 소소한 요청

### 퀴즈 문제 은행 (레슨 + 문제 세트)
- 원문을 바탕으로 카테고리당 레슨 4~7개 + 문제 20~32문항씩, 총 9~10개 카테고리
  → 약 200문항 + 레슨 60~80개 직접 작성(원문 문장을 그대로 베끼지 않고 아이 눈높이 대사/4지선다
  퀴즈+해설로 재구성)
- 카테고리: 화재원인 / 초기대응·대피 / 응급처치 / 소방시설·소화기등급 / 피난시설 /
  상황별수칙 / 자연재난 / 특별편 시나리오 / 소방관·기관 상식
- 문제 스키마에 `explain`(정답 해설) 필드 필수 포함
- 셔플백 방식으로 문제 반복 없이 순환

### 세이브/로드
- localStorage 단일 슬롯(`ffl_save_v1`), JSON (씬 위치, 퀘스트 진행, 인벤토리, 장비, HQ 티어,
  처치 몬스터, 레슨 진행도, 진급시험 응시 여부 등)

## 단계별 진행 (범위가 크므로 분할 진행)

**1단계 — 세로 슬라이스: 완료.** 마을 씬 + 소방서 씬 + 던전 1개("우리 집", Part01-03 소재),
도구 3종(분말/물호스/담요) 상성, 레슨 7개(청크 배분) + 안전 수첩, 화재이벤트 2종 + 몬스터 전투
2종, 퀴즈 32문항, 챕터 진급시험, 퀘스트 체인 6스텝, 세이브/로드까지 브라우저 실동작 검증 완료.

**2단계 — 콘텐츠 확장 (다음 작업)**: 나머지 던전(지하철/학교/공연장/백화점/산불/자연재난 등,
Part 순서 원칙에 따라 순차 추가), 퀴즈 뱅크 전체(~200문항 + 레슨), 장비 5티어 전량(K급소화기/도끼/
젖은수건 등 나머지 도구), HQ 3~5티어, 메인퀘스트 전체, 사이드퀘스트(특별편 1-4), 추모관, 우리집
점검 미니게임

각 단계 완료 시 Preview 도구로 실제 브라우저 구동 후 이동/전투/퀴즈/세이브 흐름을 직접 조작해 확인한다.

## 검증 방법
- 로컬 정적 서버(`serve.ps1` + `.claude/launch.json`)로 서빙 후 브라우저에서:
  - 방향키/D-pad로 이동, 씬 전환(문 통과) 확인
  - 던전 최초 진입 시 레슨이 조금씩 나눠 스킵 불가로 노출되는지, 인카운터마다 다음 토픽이
    이어지는지 확인
  - NPC 대화 → 퀘스트 수령/납품 흐름
  - 던전 진입 → 화재이벤트 퀴즈 판정(정답/오답 모두 해설 노출 확인) → 몬스터 전투(정답/오답 분기,
    상성 적용, 해설 노출) → 보상 지급
  - 챕터(던전) 클리어 시 진급시험이 자동 트리거되는지, 결과/보상이 정확한지 확인
  - 안전 수첩에 지금까지의 레슨/해설이 누적되어 열람되는지 확인
  - 장비 구매 → 스탯 반영 확인, HQ 자원 투입 → 티어업 및 외관 변경 확인
  - 새로고침 후 저장 데이터(레슨 진행도·진급시험 응시 여부 포함) 복원 확인
